# scan-guard

目录扫描 / 404 爆破拦截插件。

## 工作原理

1. **响应后钩子 `AfterResponse`**：WAF 每次把上游响应回写给客户端时，插件基于状态码为该 IP 累加环形桶计数。
2. **计数状态码**：400 / 401 / 403 / 404 / 405（常见扫描/探测痕迹）。
3. **触发阈值**：**10 秒内 30 次**即拉黑该 IP。
4. **拉黑时长**：600 秒（10 分钟）。拉黑期间，该 IP 所有请求在 `before_waf` 位置直接返回 403 + 统一拦截页，**不进入后续 WAF / CC / 回源流程**，成本极低。
5. **白名单**：`127.0.0.0/8`、`::1`、`10/8`、`172.16/12`、`192.168/16`、`fc00::/7`、`fe80::/10` 不计入、不拦截。
6. **IPv6**：按 `/64` 聚合，避免攻击者刷地址。

## 性能

- 热路径（`Handler`）：一次 `sync.Map.Load` + 一次 `atomic.Int64.Load`，约 30–50 ns。
- 响应后（`AfterResponse`）：若状态码不在统计集里直接 return；命中时全部 atomic 操作，无锁无分配。
- 内存：LRU 上限 10 万 IP，条目结构约 140 字节，满载约 14 MB。30s 扫描一次空闲条目回收。

## 配置（编译期常量，如需调整请改 `main.go` 然后重新编译 zip 上传）

| 常量 | 默认 | 说明 |
|---|---|---|
| `windowSec` | 10 | 滑动窗口秒数 |
| `threshold` | 30 | 窗口内错误响应阈值 |
| `blockSec` | 600 | 拉黑时长 |
| `maxTrackedIPs` | 100000 | LRU 上限 |
| `ipv6Prefix` | 64 | IPv6 聚合位 |

## 注意

- 依赖 WAF 主进程的 `AfterResponse` 插件钩子（随本版本一并引入）。
- 插件自己返回的 403 **不会**被重复计数（直接在 `before_waf` 返回，主路径的 `callPluginsAfterResponse` 不会被触发）。
